# 任务复盘模板

在需要结构化输出时，优先按下面四个问题组织：

## 1. 这次真正的目标是什么

- 用一句话描述任务真正要交付什么。
- 如果中途目标发生过变化，补一句说明变化点。

## 2. 哪些信息一开始就该给

- 列出缺失但本应前置提供的信息。
- 优先写会明显减少来回澄清或返工的部分。

## 3. 哪些返工来自需求不清、约束遗漏或验收标准不完整

- 明确返工发生在哪里。
- 判断它更像是目标不清、范围漂移、约束遗漏，还是验收标准缺失。

## 4. 下次是否值得模板化、脚本化或 skill 化

- 如果只是一次性经验，给一句下次提醒即可。
- 如果已经反复出现，明确建议落到模板、脚本、全局提示词、record skill，还是新的独立 skill。

## 输出建议

- 先给 4 个问题的简答。
- 如果有必要，再补“建议动作”。
- 只有在模式重复稳定时，才建议更新 `skill-backlog.md`。
