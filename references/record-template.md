# AI 工作记录模板

```markdown
# <线程主题>

- 日期：YYYY-MM-DD
- 来源：Codex / ChatGPT / 其他
- 类型：记录 / 回顾 / skill 候选 / skill 优化 / 代码任务 / 文档任务
- 相关目录：
- 相关 skill：
- 标签：

## 任务输入摘要

- 最终结果：
- 现有素材：
- 明确约束：
- 完成标准：
- 产出后动作：

## 背景

这次对话要解决什么问题，为什么重要。

## 关键过程

- 采取了哪些步骤。
- 做过哪些重要判断。
- 遇到哪些阻塞或返工。

## 结果

- 完成了什么。
- 修改了哪些文件或产物。
- 跑过哪些验证。

## 任务后复盘（按需）

- 这次真正的目标是什么：
- 哪些信息一开始就该给：
- 哪些返工来自需求不清、约束遗漏或验收标准不完整：
- 下次是否值得模板化、脚本化或 skill 化：

## 可复用经验

- 哪些流程下次可以复用。
- 哪些规则、脚本、模板或检查清单值得沉淀。

## Skill 观察

- 是否出现新 skill 候选：
- 是否应该优化已有 skill：
- 触发条件或典型用户说法：

## 后续事项

- [ ] 
```

## 摘要原则

- 默认写摘要，不写完整对话。
- `## 任务输入摘要` 建议在线程开始或目标明确后尽早补齐；信息不全时可以先写已知部分。
- `## 任务后复盘（按需）` 只在用户明确要求复盘，或本次工作确实暴露出明显返工点时填写。
- 保留能帮助未来工作的上下文，不保留无关闲聊。
- 不记录密钥、令牌、账号密码、隐私原文、公司敏感原文。
- 对不确定事实标注“不确定”或“需复核”。
- 记录过多时，保留有决策、踩坑、验证、复用价值的记录；低价值、重复、临时测试记录可以在确认后删除。
