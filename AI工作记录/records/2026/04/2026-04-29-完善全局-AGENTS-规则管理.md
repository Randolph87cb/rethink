- 自动化网页操作记录显示，真实站点流程应保留 dry-run、登录等待、失败调试产物和实跑验证。

- 启动任务时先读取当前项目 `AGENTS.md` 和最相关文档，再提问或修改。

- PowerShell 环境下使用兼容命令；不默认依赖 `rg`；避免 bash 风格连接符。
- 统一 UTF-8 读写；含中文 `.ps1` 需兼容 Windows PowerShell 时保存为 UTF-8 with BOM，或优先使用 PowerShell 7 / 当前会话直接执行。
- 项目有统一验证脚本时优先使用；无法验证时必须说明风险。
- 重复流程优先沉淀为项目内 skill；修改 skill 后运行 `quick_validate.py`。
- Git 操作串行；提交和推送是否默认执行，应由项目规则或用户明确决定，不建议全局默认强制。

### 全局规则候选

- 任务开始时先定位并阅读当前目录或上级目录的 `AGENTS.md`，再读取最相关的一份项目文档或历史记录；不要在未读上下文时提问或修改。
- 修改前先确认目标、范围、输出位置、格式、必须遵守和禁止使用的约束；如果用户只是讨论方案，不要提前改文件。
- 修改范围默认只覆盖当前任务相关文件；跨目录、根目录规则、导航、目录职责变化，需要先说明影响范围。
- 复杂生成任务先产出可审阅的结构、计划或清单，再批量生成最终文件；不要把关键规划只藏在脚本运行时内存里。
- 项目存在统一验证脚本时优先运行统一验证入口；没有统一入口时复用现有测试框架；不能只凭静态阅读声称已验证。
- 对外部网站、后台、发布、保存、提交、删除等有副作用操作，默认先 dry-run 或明确说明，执行前确认；登录态任务不要反复刷新登录页，等待用户手动登录并明确继续。
- 临时抓取数据、调试截图、生成中间文件应放在项目内临时或 artifacts 目录；完成后清理，或在工作记录中说明保留原因。
